## Approach Summary

Implement SDD-036 in three layers, all stdlib-only:

1. **Schema layer (strict).** Extend `cli/schema_lint.py` with a `check_depends_on` validator that fires only when a `spec.md` carries `depends_on`. Add the optional inline-list field; do not touch `REQUIRED_CONTRACT_FIELDS`.
2. **Reorder layer (strict).** Add a new `cli/backlog_reorder.py` module that performs a display-order move against `backlog/display-order.json`, enforces dependency-lock (using `depends_on` + DONE-state), governs force-override, and appends one row to `ledger/reorder-audit.jsonl`.
3. **Render layer (UI-variant).** Extend `cli/state_builder.py`'s HTML rendering to draw the lifecycle pipeline, the four-card docs row (with missing-state), and a keyboard-accessible reorder control that reflects the display-order overlay.

Each layer is paired with tests. The render layer opts into the UI Lifecycle Variant so visual REQUIREDs can be refined via append-only deltas during F-25; the schema and reorder layers stay strict Article X.

## Parallel-Safe Tasks

- None are safely parallel across workers. `cli/state_builder.py` and `cli/schema_lint.py` are **shared single files**, and the reorder module shares the `depends_on` inline-list parsing convention with both. Two workers editing either shared file concurrently would conflict.

## Sequential Tasks

- All F-25 tasks run **serially in one session**. The kickoff is explicit: "Shared `cli/state_builder.py`, `cli/schema_lint.py`, generated exec surfaces, or SDD-036 artifacts force serialization." Recommended order: P1 (schema) -> P2 (reorder) -> P3 (render) -> P4 (verify), because the reorder layer depends on the `depends_on` parsing settled in P1, and the render layer reads both the overlay (P2) and `depends_on` (P1).

## Serialization Verdict

**Fleet dispatch is NOT safe for SDD-036; serialization is forced.** The only genuinely new files (`backlog_reorder.py` + its test) still serialize behind the shared `depends_on` parsing decision and behind shared edits to `schema_lint.py`/`state_builder.py`. F-25 must run as a single ordered session.
