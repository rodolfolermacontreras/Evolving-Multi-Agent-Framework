## Scott Feedback Bundle (captured 2026-06-03)

Source: 1h43m meeting transcript with Scott Epperly (Senior DS, Software Architecture + Data Engineering, Microsoft WWIC), 2026-06-02. Scott runs a parallel similar system on Incentive Analytics and provided concrete patterns from production usage. All items below are extracted from that conversation and routed here for PM triage as one coherent epic.

- **[2026-06-03]** Portability via `.github/` symlink trick -- "host repo's root `.github/` is in .gitignore (committed empty), each project ships its own `.github/` inside its project folder, a bootstrap script creates symlinks from `<repo-root>/.github/agents` -> `<project>/.github/agents` (and `skills/`, `instructions/`, `prompts/`). VS Code sees the symlinked content, git sees nothing, no pollution of host history, no collisions across developers on the same branch." -- Extend `cli/bootstrap.py brownfield` to add `.github/` to host `.gitignore`, emit a PowerShell + Bash symlink script, and add a new operational skill `host-integration-symlink`. Directly unblocks PI-3 brownfield portability test. -> Route to PM (P1, blocker for PI-3).

- **[2026-06-03]** Serial gate on CLARIFY and SPEC across the whole backlog -- "in classical BA + Tech Spec work, requirements drafting is serial for a reason. The drafter must hold all in-flight specs in their head to spot conflicts. Atomizing per-feature destroys that cross-check. Only one work item can be in story-drafting or spec-drafting at any time, repo-wide. Implementation is parallel; clarification/specification is not." -- Amend constitution (refinement of Article VI or new sub-article under VII). `fleet.py status` flags violations. Architect/PM agent prompts gain a precondition: before authoring a spec, scan all in-flight specs and recent DONE backlog for overlap/conflict, write the finding into `clarification-log.md`. -> Route to PM + Architect (P1, complements the new pre-work-check skill).

- **[2026-06-03]** Cross-feature deduplication pass at triage and clarify -- "three separate user-reported fixes came in over two weeks and the agent created three separate features. A human immediately saw they were one underlying issue. The agent missed it because each report was atomic." -- Mandatory pre-spec step in `/triage` and `/clarify`: search BACKLOG.md, all `specs/*/spec.md`, and recent retros for semantic overlap with the incoming idea. Emit a "potential-duplicate" block in the triage output. Require PM/Architect to explicitly acknowledge or refute before proceeding to SPEC. Synergistic with the pre-work-check skill above. -> Route to PM (P1).

- **[2026-06-03]** End-of-session self-review loop -- "when I finish a feature, I flip back into a generic agent session and ask the agent to review the session itself for inefficiencies (repeated tool failures, missing instructions, redundant calls) and propose amendments back to the agent definitions." -- New skill `session-self-review` under `operational/`. Runs at `/replan` time and produces: list of repeated errors or retry loops detected in the session, proposed deltas to specific agent definitions or skills, routed to `/evolve` as candidate lessons. Formalizes a meta-learning loop we are doing manually today. -> Route to PM (P2).

- **[2026-06-03]** UI development lifecycle variant (relaxed Article X for visual work) -- "AI writing HTML and thinking it got it right never works. UI is inherently iterative and visual. The strict spec-first lifecycle breaks down. Considering pulling UI work into a separate sub-loop where the spec gets updated by the developer during iteration, rather than locked at TASKS." -- New ADR or article amendment: UI features get relaxed Article X validation contract that may mutate during IMPLEMENT, with each mutation recorded in `validation.md` as a delta entry. Alternative: separate `/ui-implement` slash command with UAT-loop state machine. The current dashboard work has hit this exact pain and is the right pilot. -> Route to Architect (P2).
