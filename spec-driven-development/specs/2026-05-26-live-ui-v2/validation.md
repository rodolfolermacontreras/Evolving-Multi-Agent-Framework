## 3. Definition of DONE

The Live UI v2 implementation is DONE when ALL of the following are true:

1. **All IAC criteria pass (IAC-01 through IAC-15)**
   - Verified by automated tests in `test_state_builder.py`
   - Each IAC maps to at least one test case

2. **All VAL criteria pass (VAL-01 through VAL-21)**
   - VAL-19 requires manual screen reader verification (HITL)
   - All others are automated

3. **Test suite health**
   - Full test suite passes: `pytest cli/test_state_builder.py -v --tb=short` exits 0
   - Test count >= pre-implementation baseline + 30 new tests (estimated from tasks.md)
   - No test uses real filesystem outside `tmp_path`
   - No test makes network requests

4. **Code quality (Stage 2 review)**
   - No bare `except:` blocks
   - All dynamic HTML values pass through `h()` (html.escape)
   - No magic numbers/strings (extract to constants or design tokens)
   - No commented-out code, unused variables, or orphan imports
   - No inline styles in generated HTML (all styles via CSS classes)
   - Commit format: `type: short description`

5. **Visual fidelity**
   - Generated `state.html` visually matches `mockup.html` at 1280px and 768px widths
   - Verified by human inspection (HITL)

6. **No regressions**
   - `render_markdown()` output is identical for same inputs (SDD-002 contract intact)
   - `build()`, `serve`, `build-index` commands work correctly
   - `--dry-run` and `--json` flags work correctly

7. **Integration**
   - Feature branch merged into `integration/improvements`
   - Full test suite passes on `integration/improvements` post-merge
   - BOARD.md updated: feature marked DONE with completion date
